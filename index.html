If you are using Safari 8 on Yosemite, you might have noticed by now.
The big blue header is not visible behind the blurred "menu bar" in safari. Instead, you see a nice rainbow.

Why?

You can use this to hide things (as explained below), but also to brand safari to your own brand colours, use the menu bar as a status indicator or loading bar, use it to get your users attention, use it to look totally hipster with a custom menu bar, or use it just because you like it.
With a little more javascript, a lot more should be possible.

The problem

I was annoyed by "hidden menus" not being fully hidden on Safari 8. If you look at for example a medium.com article, and scroll down, the menu at the top of the page moves up (disappears) and shows again when you scroll down. But in Safari 8, it does not fully disappear, as its still visible behind the blurred background.

The solution

The idea to solve this was to add an overlay top of the page, with the same color as the website background color, to hide everything that was supposed to scroll behind it. But then came up with an even better idea. The background of the overlay could be any color. Yes, any. Thats when RainbowSafari was created.

Get it

Below is all the code you need (except for jQuery, you need that too, sorry). Add the CSS to your CSS file and put the javascript between script tags in the head of your page. You do not have to use a single background color. Images (even animated) and gradients are also supported.

Javascript

var isSafari = Object.prototype.toString.call(window.HTMLElement).indexOf('Constructor') > 0;
$( document ).ready(function() {
  if (isSafari) {
      window.onscroll = function () { 
        var doc = document.body, 
        scrollPosition = doc.scrollTop
          $( "#rainbowsafari" ).height( 100 ).css({
            top: scrollPosition - 100
          });
       }; 
   }
});

CSS

#rainbowsafari {
    background-color: #FFFFFF; /* Edit This */
    position: absolute;
    left: 0;
    right: 0;
    top: 0;
    height: 0px;
    z-index: 99999999999;
}

Problems

As it uses javascript to get the scroll position, RainbowSafari sometimes lags behind a little bit. Especially when using trackpads with natural scrolling. If you have a solution for this, please contribute!
Also, gifs only play when scrolling. Beware of that.
